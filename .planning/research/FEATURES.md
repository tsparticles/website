# Research — Features

Scope

- Table-stakes and differentiating features for a developer-focused documentation and demo site like tsParticles.

Table stakes (what users expect)

- Clear API reference generated from source (TypeScript) — `docs-gen/` must be reproducible and up-to-date.
- Searchable docs and navigation — users must find API and examples quickly.
- Interactive demos with editable configuration — live examples that reflect `configs/`.
- Copyable configuration snippets (JSON) and clear usage examples.

Differentiators (competitive)

- Playground with importable presets and visual feedback — let users experiment with configurations and export them.
- Versioned docs with changelog and migration guides — useful if library evolves.
- Performance tips and common pitfalls section — targeted at developers integrating particles efficiently.

Anti-features (deliberately avoid)

- Heavy client-side frameworks for the site UI — keep site lightweight.
- Requiring user accounts to access docs or demos.

Confidence

- Medium — based on common patterns for developer documentation sites and repo contents.
